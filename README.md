# Real-Time Ticketing Frontend

실시간 티켓팅 서버와 연동되는  
Vue 기반 티켓팅 프론트엔드 포트폴리오입니다.

대용량 티켓팅 환경에서의  
좌석 선점 → 결제 → 확정 흐름을  
프론트에서 구현했습니다.

------------------------------
# 프로젝트 목적

티켓팅 시스템의 핵심인

- 동시성 상황 UX
- 좌석 선점 흐름
- 결제 타이머
- 직접 URL 접근 차단

을 프론트엔드에서 구현하고  
백엔드(분산 서버 구조)와 연동하기 위한 포트폴리오입니다.

------------------------------
# 아키텍처
 
Client (Vue)
↓
ALB
↓
Spring Server x2
↓
Redis (좌석 선점)
↓
RDS (예약 저장)

* 프론트는 ALB 뒤의 다중 서버 환경을 가정하고 구현했습니다.

------------------------------
# 구현 화면

### 1️⃣ 이벤트 화면
- 티켓팅 시작 버튼
- 플로우 기반 접근 시작

### 2️⃣ 좌석 선택 화면
- 좌석 클릭 시 서버 hold API 호출
- 동시성 상황 대응 UX

### 3️⃣ 결제 대기 화면
- 120초 타이머
- confirm API 호출

------------------------------
# 직접 URL 접근 차단

SPA 구조에서  
사용자가 직접 URL 입력하거나  
새로고침 시 플로우가 깨지는 문제를 방지하기 위해

```

router.beforeEach
sessionStorage gate 방식

```

으로 특정 페이지 접근을 제어했습니다.

차단 대상:
- /seat
- /checkout

허용 방식:
- 버튼 클릭으로만 진입
